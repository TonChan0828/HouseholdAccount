## 決めたこと・理由

- `/demo` 配下に完全クライアントサイドの並行ルートツリーを新設する方針。既存の Server Component / Server Action / Supabase には手を入れず、React Context のインメモリ状態を真実源とする（リロードで初期化＝要件「永続化しないがリロードまで保持」を自然に満たす）。
- 末端コンポーネント・フォーム（`TransactionForm`/`CategoryForm`）・純粋ヘルパー（period/category-matrix/format）・Zodスキーマを再利用し、重複はページ上部レイアウトのみに留める。
- 対象画面: 収支CRUD＋カテゴリCRUD＋ダッシュボード。初期データ: サンプル投入。導線: 常時バナー＋登録ボタン。
