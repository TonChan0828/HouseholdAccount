## 決めたこと・理由

- **2カラム化はダッシュボードのみ**: 収支一覧・フォーム系は縦1カラムが読みやすく、
  一覧性の課題が顕著なのはホームのみのため（他ページは仕様の未解決課題に記載）
- **`contents` + `max-lg:order-*` 方式**: モバイルの表示順・見た目を一切変えずに
  デスクトップだけ2列へ再配置でき、grid-template-areas の行高さ連動問題も回避できる
- **サイドレールに予測・目標・最近の取引**: 情報量の少ないグランサブルなカードを
  細い列へ寄せ、幅を要するヒーロー・チャート・マトリクスをメイン列に置く
- E2E セレクタ（data-testid・見出し・リンク名）は不変のため既存 E2E で回帰を担保
