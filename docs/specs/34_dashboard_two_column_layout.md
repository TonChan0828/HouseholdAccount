# ダッシュボード2カラム構成（デスクトップ） 仕様書

## 概要

ダッシュボード（/dashboard・/demo/dashboard）はカード7ブロックを最大幅 896px（max-w-4xl）の
1カラムに縦積みしており、デスクトップでは左右に大きな余白が生まれ、全体を見るのに深い
スクロールが必要だった。

本機能では、デスクトップ（lg 以上）でダッシュボードを「メイン列 7 : サイドレール 5」の
2カラムグリッドに再構成し、一覧性（ファーストビューに収まる情報量）とレイアウトの
美しさを両立する。**モバイル（lg 未満)の表示順・見た目は現状を維持する。**

## 対象ユーザー・前提条件

- ログイン済みで家計簿グループを選択済みのユーザー（/dashboard）
- デモモード利用者（/demo/dashboard）
- データ取得・集計ロジック・カード内部の UI は変更しない（純粋なレイアウト再構成）

## 画面・UI

### 表示内容

- コンテナ幅: `max-w-4xl` → `lg:max-w-6xl`（アプリヘッダーの max-w-6xl と揃える）
- デスクトップ（lg 以上）は 12 カラムグリッドで 2 カラム構成にする

| 列 | 幅 | /dashboard | /demo/dashboard |
| --- | --- | --- | --- |
| メイン列 | 7/12 | 収支ヒーロー → 当期の収支チャート → メンバー別カテゴリ | 収支ヒーロー → 当期の収支チャート → メンバー別カテゴリ |
| サイドレール | 5/12 | 月末着地予測 → 貯金目標 → 最近の取引 | 最近の取引 |

- モバイル（lg 未満）は現行どおり 1 カラム。表示順も現行を維持する
  - /dashboard: ヒーロー → 着地予測 → 貯金目標 → チャート → マトリクス → 最近の取引
  - /demo/dashboard: ヒーロー → チャート → マトリクス → 最近の取引

### インタラクション・バリデーション

- 実現方法: 共有プリミティブ `MainSideGrid`（`components/shared/main-side-grid.tsx`。
  当初 `DashboardGrid` として dashboard 配下に置いたが、カレンダー（spec 29）でも
  使うため shared へ移動・改名した）
  - 外側: `grid grid-cols-1 gap-5 lg:grid-cols-12 lg:items-start lg:gap-6`
  - main / side ラッパー: モバイルでは `contents`（子が外側グリッドに直接参加し、
    `max-lg:order-*` で現行の縦積み順を再現）、lg 以上では `lg:block lg:space-y-6` の列になる
  - 列は `items-start` で高さを独立させ、片方が短くても間延びしない
- 既存のフェードイン演出（reveal）・アニメーション遅延は維持する
- E2E のセレクタ（data-testid・見出しテキスト・リンク名）は変更しない

## 未解決の課題

- メンバーが多い（5人以上）グループではメイン列のマトリクスが横スクロールになる
  （既存の sticky 先頭列 + スクロールヒントで対応済みだが、列幅の自動間引きは今後の検討）
- 収支・分析など他ページのデスクトップ構成は今回のスコープ外（必要なら別仕様で検討）
