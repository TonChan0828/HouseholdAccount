# UIオーバーホール 仕様書

## 概要

全機能（00〜07）実装済みのアプリに対し、フロントエンドを全面的に磨き込む。
現状は shadcn デフォルトの無彩色テーマ・共通ナビなし・アイコンなしの素朴な UI であり、
「温かみのある家計簿」をコンセプトにテーマ・アプリシェル・各ページの UI を刷新する。

機能追加ではなく見た目と操作感の改善であり、DB スキーマ・Server Action のロジックは変更しない。

## 対象ユーザー・前提条件

- 既存の全ユーザー（ログイン済み・グループ選択済みのメンバー）
- 既存機能の動作・データフローは変更しない（既存テストの `data-testid`・主要文言は維持する）
- 対応テーマはライトのみ（CSS 変数はダークテーマを将来追加できる構造にする）

## デザインコンセプト

- **テイスト**: 温かみのある家計簿。家族で使う親しみやすさを優先する
- **配色**: クリーム系の背景（`oklch` で低彩度の暖色）+ 純白カード + 深いグリーンの primary。
  収入/支出は `--income` / `--expense` のセマンティック変数で統一する（Tailwind クラス直書きを廃止）
- **形状**: 大きめの角丸（`--radius` ≒ 1rem）と柔らかい多層シャドウ
- **フォント**: 見出しは丸ゴシック（M PLUS Rounded 1c）、本文は Noto Sans JP
- **アイコン**: lucide-react をナビ・サマリー・空状態に使用する

## 画面・UI

### Phase 1 — デザイン基盤 + アプリシェル

- `app/globals.css` のテーマ変数を上記コンセプトで刷新する
- `app/layout.tsx` に `next/font/google` でフォントを導入する
- アプリシェル（`components/features/layout/`）を新設し `(dashboard)/layout.tsx` に組み込む
  - **デスクトップ**: スティッキーヘッダー（ロゴ + アイコン付きナビ + 現在のグループ名 + ユーザーメニュー）
  - **モバイル**: 下部固定タブバー + 中央の「収支を記録」FAB
  - ナビ項目: ダッシュボード / 収支 / 分析 / カテゴリ / メンバー / グループ設定
  - 現在ページをハイライト表示する
- 各ページ先頭のボタン列ナビ（`buttonVariants` の Link 並び）を削除する
- コンテンツ幅をページ性質に応じて `max-w-4xl`〜`5xl` に拡張する

### Phase 2 — ダッシュボード + 収支

- サマリーを収入/支出/収支の 3 枚カードに分離（アイコン・大きな数字・収支のポジ/ネガで色が変わる）
- 取引リスト: 日付グルーピング見出し + カテゴリ色バッジ + ホバー演出 + アイコン付き空状態
- 取引フォーム: 収入/支出の大きなセグメント切替、金額入力の強調、カテゴリの色チップ選択

### Phase 3 — 残りページ + 仕上げ

- 分析: チャート配色をテーマパレットに統一しカード化
- メンバー: イニシャルアバター + メンバーごとのアクセントカラー
- カテゴリ: 色スウォッチを前面に出したカードグリッド
- グループ選択: カード選択 UI（アクティブグループの強調）
- 認証画面: ブランドパネルで第一印象を改善
- tw-animate-css による軽い出現アニメーション

### インタラクション・バリデーション

- フォームのバリデーションルールは既存仕様（02/04/06）から変更しない
- ナビのアクティブ状態は `usePathname` で判定する
- モバイルタブバーは `(dashboard)` ルート群でのみ表示する

## データモデル

UI のみの変更で新規の型・テーブルは追加しない。既存の `Transaction` / `Category` / `Household` / `Profile`（`types/index.ts`）をそのまま使用する。
