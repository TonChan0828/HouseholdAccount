# 金額の四則演算入力 仕様書

## 概要

収支登録フォームの金額欄で**四則演算式**を入力し、計算結果を金額として登録できるようにする機能。
レシートの合計（例: `1280+980+550`）や割り勘（例: `3000÷4`）を、電卓を使わずその場で計算できる。

金額は円なので整数で保存する。式の計算結果に小数が出た場合は**四捨五入**で整数に丸める。

## 対象ユーザー・前提条件

- ログイン済みかつアクティブな household に所属しているユーザー。
- 対象は収支の新規登録フォーム・編集フォームの両方（同一の `TransactionForm`）。
- マイグレーション不要（`transactions.amount` は従来どおり整数）。

## 評価仕様

評価ロジックは純粋関数 `lib/amount-expression.ts` に集約し、クライアント（プレビュー）と
サーバー（zod スキーマ）で共用する。`eval` / `Function` は使わず、自前のトークナイザ＋
再帰下降パーサで実装する。

- **対応演算子**: `+` `-` `*` `/` と括弧 `(` `)`。優先順位は数学どおり（`*` `/` が優先、括弧で上書き）。
- **正規化**: 入力をパース前に正規化する。
  - 全角→半角: 数字 `０-９`、記号 `（）＋－×÷．`
  - `×` → `*`、`÷` → `/`（`*` `/` も受理）
  - 空白を除去
- **単項符号**: 先頭やカッコ直後の `-` `+`（例: `-500`、`(-3)*2`）を許可。
- **小数**: 入力途中の小数（例: `1000.5`）も計算可。最終結果のみ呼び出し側で丸める。
- **無効** → `null` を返す:
  - 構文エラー（例: `1000+`、`abc`、`1+*2`、括弧不整合）
  - 0除算、結果が非有限（`Infinity` / `NaN`）

### API
- `evaluateExpression(raw: string): number | null` — 生の計算結果（小数含む）または `null`。
- `evaluateAmount(raw: string): number` — `evaluateExpression` を `Math.round`。無効なら `NaN`。

## 画面・UI

- 金額 `Input` を `type="number"` → `type="text"`（`inputMode="text"`）に変更し演算子を入力可能にする。
  `name="amount"` / `required` / `¥` プレフィックスは維持。
- 入力欄の下に計算結果を**リアルタイムプレビュー**（`role="status" aria-live="polite"`）:
  - 空、または演算子を含まないプレーン数値 → 非表示
  - 演算子を含み有効 → `= ¥1,500` のように丸め後の金額を表示（`lib/format.ts` の `yen()` で整形）
  - 演算子を含むが無効 → 「計算できません」を控えめな色で表示
- 「登録して続ける」成功時のクリア処理でプレビュー表示もリセットする。

## バリデーション

- 検証は**サーバー側（zod）を正**とする。フォームは式の文字列をそのまま送信する。
- `lib/validations/transaction.ts` の `amount` を preprocess に置き換え:
  - 文字列なら `evaluateAmount` で評価・丸め（無効は `NaN`）。
  - `z.number().int().min(1)` で検証（非整数・1円未満・`NaN` はエラーメッセージで拒否）。
- 既存のエラーメッセージ（「金額は整数で入力してください」「金額は1円以上で入力してください」）を踏襲。
  評価不能な式には「金額を正しく入力してください」を返す。

## 未解決の課題

- モバイルで演算子入力用のオンスクリーンキーは設けない（OSキーボードに委ねる）。将来 UX 改善余地あり。
- パーセント・指数などの拡張演算子は対象外（YAGNI）。
